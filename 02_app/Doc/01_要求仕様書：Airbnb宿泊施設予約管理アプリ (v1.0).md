# 要求仕様書：Airbnb宿泊施設予約管理アプリ (v1.0)

## 1. プロジェクト概要
本システムは、LINE公式アカウントを窓口とし、Vercel・GitHub・Googleスプレッドシート・GAS（Google Apps Script）を連携させた、Airbnb宿泊施設の予約管理アプリケーションである。リピーター客に対して予約カレンダー案内・空き状況・定期的な予約のおすすめをLINEから通知し、問い合わせ・予約希望の受付から確定通知までを一貫して管理する。

---

## 2. システム構成・アーキテクチャ
フロントエンドとバックエンドを分離し、LINE Messaging API（Webhook）およびLINE Login を組み合わせた構成とする。

* **フロントエンド（管理者用）:** Vercel（HTML5 / CSS3 / JavaScript）
    * 管理者（井上）が通知メッセージの編集・設定を行うWeb画面。GitHub連携でデプロイ。
* **バックエンド:** Google Apps Script (GAS)
    * Webhook受信、スプレッドシートへのデータ保存・取得、LINE Messaging API によるプッシュ通知、定期実行（時間ベースの通知）を制御。
* **データベース:** Google Sheets（スプレッドシート）
    * 問い合わせ履歴、予約データ、ゲスト情報（LINE User ID紐付け）、通知テンプレート文面などを保存。
* **プラットフォーム:** LINE公式アカウント
    * ゲストとのメッセージの送受信、LINE Login によるユーザー識別、管理者への通知配信。

---

## 3. 役割・権限

### 3.1 ゲスト（宿泊者）
* LINE公式アカウントを友だち追加したお客様。
* LINE Login により取得した **LINE User ID** で一意に識別し、問い合わせ・予約を紐付ける。
* 予約希望メッセージの送信、予約確定通知・各種案内メッセージの受信を行う。

### 3.2 管理者（井上）
* **管理者LINE User ID** をシステムに登録し、管理者権限として扱う。
* ゲストからの問い合わせ・予約希望が届いた際、**Webhook経由で管理者のLINEアカウントに自動通知**される。
* 問い合わせ内容・ゲストのLINE User ID を把握したうえで、空き状況に応じた返信（宿泊料金・清掃代1万円の案内、予約確定の可否）を行う。
* 予約確定時は、管理者のLINEにも「予約確定」通知が届く。

---

## 4. 機能要件

### 4.1 問い合わせ・予約希望の受付
* **ゲストがLINEでメッセージを送信**すると、その内容をGASのWebhookで受信する。
* **Googleスプレッドシート**に、ゲストの **LINE User ID** と紐づけて問い合わせ内容（メッセージ本文・日時）を保存する。
* 同時に、**管理者（井上）のLINE** に、問い合わせ内容が自動で通知される（Webhook／Messaging API の送信先を管理者のLINE User ID に設定）。

### 4.2 空き状況に応じた案内・予約確定
* 管理者が日程を確認し、**該当日程が空き**の場合：
    * ゲストに「宿泊料金＋清掃代1万円」および「ご予約確定しますか？」の案内をLINEで送信する。
* ゲストが予約確定した場合：
    * ゲストに**予約確定の通知**を送信する。
    * **管理者のLINE** に、予約確定の旨（ゲストのLINE User ID や予約内容の要約を含む）が通知される。

### 4.3 予約確定後の自動通知（ゲスト向け）
以下のタイミングで、ゲストのLINEに自動通知する。文言・URLはスプレッドシートまたはVercel管理者画面で管理する。

| タイミング | 通知内容 |
| :--- | :--- |
| **予約確定時** | 周辺のおすすめレストラン案内 ＋ **キャンセルポリシー**（チェックイン日から逆算した具体的な日付と料金率を自動計算して表示） |
| **チェックイン3日前** | チェックインガイドPDFのURLを送信 |
| **宿泊日の1週間前** | 宿泊1週間前の案内メッセージ（文言はVercel管理者画面で編集可。編集結果はスプレッドシートに保存し、この通知で使用） |
| **チェックイン当日・お昼頃** | 玄関ドアの開け方動画（YouTube URL）を送信 |
| **チェックイン当日・夕方4時頃** | 「チェックイン完了できましたか？」の案内・チェックイン補足 |
| **チェックアウト前日・お昼頃** | 「明日はチェックアウトです。お忘れ物のないようご注意ください。清掃へのご協力をお願いします」等の案内 |

**キャンセルポリシーの詳細:**
- 1週間前（7日前）：50%
- 4日前：80%
- 3日前：100%
- キャンセル料は清掃代も含む
- 予約確定時のLINEメッセージに、チェックイン日から逆算した具体的な日付（例：「3月10日以降のキャンセル料は50%」）を自動計算して表示

### 4.4 管理者画面（Vercel）と管理者専用設定
* **アクセス制御:** Vercel の設定画面・集計画面は **LINE Login（LIFF）で開き、取得した LINE User ID が管理者 LINE User ID と一致する場合にのみ表示・編集可能**とする。宿泊ゲストが LINE から同じ LIFF にアクセスした場合は、**設定項目は一切表示せず**、カレンダー予約画面などゲスト向けの画面のみ表示する。
* **管理者の LINE でアクセスした時のみ編集できる項目:**
  * **Google Drive フォルダID** … 月間売上 PDF の保存先。管理者画面でのみ設定・変更可能。
  * **返信メッセージの編集** … 問い合わせ等への返信用テンプレート。
  * **予約確定メッセージ** … ゲストに送る予約確定時の御礼メッセージ文面。
  * **一泊単価（価格設定）**
  * **清掃代（価格設定）**
  * **ペット宿泊ありの場合の清掃代（価格設定）**
  * **周辺のおすすめレストラン案内** … 予約確定時にゲストに送る文章。
  * **チェックインガイド PDF の URL** … チェックイン3日前に送信する URL。
  * **宿泊1週間前の案内メッセージ**
  * **玄関ドアの開け方動画の URL** … チェックイン当日お昼に送信する YouTube 等の URL。
  * **リッチメニュー用メッセージ** … 施設情報、利用規約、メッセージで問い合わせ誘導文、アクセス案内、ゴミ捨て方法。
  * **Airbnb iCal URL** … 空き状況取得元。
  * **GoogleカレンダーID** … LINE予約をAirbnbに同期するためのカレンダーID。
  * **清掃員LINEグループID** … 予約確定時に通知を送るグループID。
  * **予約禁止期間設定** … 管理者が任意の期間を設定し、LINEアプリ上でのみ予約をブロック（Googleカレンダー・Airbnbには影響しない）。
* 上記はいずれも **管理者の LINE ID でアクセスした時のみ** Vercel 画面上で編集でき、編集結果は Google スプレッドシート（config / templates / blocked_periods）に保存する。

### 4.5 月間売上集計と PDF 保存（Google Drive）
* **トリガー:** 各月が締まったタイミング（月末締めまたは月初に前月を締める運用）で、その月の売上を集計する。
* **集計内容:** 確定予約（reservations の status＝確定）のうち、チェックイン日（または確定日）が対象月に含まれるものを集計。売上合計・件数・内訳（宿泊料金合計・清掃代合計など）を算出する。
* **PDF 出力:** 集計結果を PDF ファイルにし、**管理者が設定した Google Drive のフォルダID** に保存する。ファイル名は例として「月間売上_YYYY年MM月.pdf」とする。
* **フォルダID の設定:** 月次 PDF 用の Drive フォルダID は、**管理者の LINE でアクセスした時のみ** Vercel の管理者画面で設定・変更できる（上記 4.4 に含まれる）。

### 4.6 予約確定時の清掃員への通知（LINE グループ／オープンチャット）
* **要件:** 予約確定時に、清掃員用の **LINE オープンチャット（またはグループトーク）** にも予約確定の自動通知を流したい。
* **可否:** **可能**。LINE Messaging API の Push は、ユーザーIDに加え **グループID** 宛にも送信できる。清掃員用のオープンチャット（またはグループ）に **公式アカウントのボットをメンバーとして追加**し、そのトークで発言した際などにイベントから取得できる **グループID** を config に保存する。予約確定処理の一環で、管理者の LINE への通知と同様に、**そのグループID宛にプッシュメッセージ**（予約内容の要約）を送信する。
* **注意:** オープンチャットでグループIDが取得・利用できるかは LINE の仕様に依存する。通常のグループトークであれば問題なく利用可能。オープンチャットの場合は、ボットを追加したうえでグループIDが取得できる場合に本機能を利用する。

### 4.7 予約キャンセル機能
* **要件:** ゲスト（および管理者）が自分の予約をキャンセルできる機能。
* **キャンセル料計算:**
  - チェックイン日の7日前まで：無料（0%）
  - チェックイン日の7日前〜4日前：50%
  - チェックイン日の4日前〜3日前：80%
  - チェックイン日の3日前以降：100%
  - キャンセル料は清掃代も含む
* **キャンセル確認:** キャンセルボタンをクリックすると、「本当にキャンセルしますか？キャンセル料＝￥〇〇〇円です。」という確認ダイアログを表示。
* **処理:** キャンセル確定後、予約のステータスを「キャンセル」に更新し、キャンセル料を備考欄に記録。管理者とゲストにLINE通知。Googleカレンダーからも該当予約を削除。

### 4.8 予約禁止期間設定機能
* **要件:** 管理者が特定の期間を予約禁止に設定できる機能。メンテナンス、プライベート利用、改装工事などの理由で、一時的に予約を受け付けたくない期間を設定。
* **設定方法:** 管理者画面で開始日・終了日・理由（オプション）を入力し、複数の期間を追加可能。
* **影響範囲:** **LINEアプリ上でのみ予約をブロック**。Googleカレンダーには追加されず、Airbnbにも同期されない。
* **カレンダー表示:** 予約禁止期間はグレーアウト（選択不可）として表示。
* **削除:** 一覧から「削除」ボタンで期間を削除可能。

### 4.9 権限の分離
* **ゲストのLINE User ID** と **管理者（井上）のLINE User ID** をシステム上で区別する。
* 管理者用の処理（問い合わせ通知の受信、予約確定の受信）は管理者のLINE User ID にのみ送信する。
* ゲスト用の各種案内は、予約に紐づくゲストのLINE User ID に送信する。
* Vercel の設定画面は管理者の LINE ID でアクセスした時のみ表示し、ゲストがアクセスした場合は表示しない。

### 4.10 Airbnb iCal 連携・カレンダー予約（Webサイト）
* **Airbnb カレンダー連携:** Airbnb が提供する iCal 形式のカレンダーURL（例: `https://www.airbnb.jp/calendar/ical/20183082.ics?t=...`）をシステムに登録し、**空き状況の取得・同期**を行う。iCal には既に予約が入っている日付（ブロック日）が含まれるため、これをパースして「予約済み＝選択不可」「空き＝選択可能」として扱う。
* **Vercel 上のカレンダー画面:** ゲスト向け（または管理者・ゲスト共通）の Web ページで、**カレンダー形式**で空き状況を表示する。表示データは上記 iCal を元に算出する。
* **予約条件の選択:** 画面上で以下を選択できるようにする。
    * **チェックイン日**（カレンダーから選択、空き日のみ選択可能）
    * **チェックアウト日**（チェックイン以降の空き日から選択）
    * **宿泊人数**（プルダウンまたは数値入力）
* **総支払金額の表示:** 選択した宿泊日数・人数に応じて、**宿泊料金（1泊あたり料金×泊数）＋清掃代（1万円）** を計算し、画面上に「総支払金額」として表示する。料金ルール（1泊単価・人数による加算など）は設定（config）で管理する。
* **宿泊確定ボタン:** 「この金額で宿泊を確定させる」のようなボタンを配置する。押下時に、選択内容（チェックイン・チェックアウト・人数・金額）をバックエンドに送信し、予約を確定する。
* **予約確定時の自動処理:**
    1. **ゲストのLINE:** 予約確定の御礼メッセージ（管理者が設定した文面）を自動送信する。ゲストは **LINE Login（LIFF）** でカレンダー画面にアクセスしている前提とし、取得した LINE User ID に紐付けて予約を保存し、その LINE に御礼メッセージを送る。
    2. **管理者のLINE:** 「予約が入りました」という通知（ゲスト名・日程・人数・金額の要約）を管理者（井上）の LINE に送信する。
    3. **清掃員のLINE グループ／オープンチャット:** 管理者が config に登録したグループID（清掃員用トーク）宛に、予約確定の通知（日程・チェックアウト日等の要約）をプッシュ送信する（4.6 参照）。
    3. **Airbnb カレンダーへの反映:** Airbnb の iCal URL は**エクスポート（読み取り）専用**のため、当システムから直接 iCal で Airbnb に書き込むことはできない。確定した予約を Airbnb のカレンダーに反映する方法は次のいずれかとする。
        * **推奨:** 管理者が Airbnb の管理画面（予約・カレンダー）で、該当期間を手動でブロックする。
        * **将来対応:** Airbnb がホスト向け API でカレンダー更新を提供している場合、その API を呼び出してブロックを追加する。
        * 当システムでは、確定予約を Google スプレッドシートに記録し、管理者通知に「Airbnb で該当日程をブロックしてください」などの案内を含めることで、二重予約を防ぐ運用をサポートする。

## 5. 非機能要件

### 5.1 セキュリティ・運用
* **HTTPS:** 全ての通信はSSL/TLSで保護する。
* **CORS:** VercelからGASへのAPI呼び出しを許可する設定とする。
* **Webhook検証:** LINE Platform からのWebhookは署名検証で正当性を確認する。

### 5.2 可用性・保守
* フロントエンドはGitHub連携によりVercelにデプロイし、変更履歴を管理する。
* GASの定期実行（時間駆動）で、上記「1週間前」「チェックイン当日」等の通知タイミングを実現する。

---

## 6. データの扱い（概要）
* **問い合わせ履歴:** ゲストのLINE User ID・メッセージ本文・受信日時をスプレッドシートに保存。
* **予約データ:** ゲストのLINE User ID・チェックイン日・チェックアウト日・予約確定日時・金額等を保存。
* **通知テンプレート:** 「1週間前」の文面など、編集可能なメッセージはスプレッドシートの専用シートまたはセルに保存し、GASから参照する。
* **管理者LINE User ID:** 設定用シートまたはスクリプトプロパティで保持する。

---

## 7. 今後の拡張（オプション）
* 予約カレンダー・空き状況の LINE 上での照会（本仕様では Vercel Web のカレンダーで対応）。
* リピーター向けの定期的な空き状況・おすすめ予約のブロードキャスト。
* 複数施設・複数管理者への対応。
* Airbnb 公式 API によるカレンダー書き込み（API 提供時に自動ブロック反映を実装）。
