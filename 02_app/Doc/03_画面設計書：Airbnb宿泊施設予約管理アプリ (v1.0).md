# 画面設計書：Airbnb宿泊施設予約管理アプリ (v1.0)

## 1. 画面設計方針
* **管理者・ゲストの分離:** Vercel の LIFF では、**LINE Login で取得した LINE User ID が config の管理者 LINE User ID と一致する場合のみ**「管理者用設定画面」を表示する。**宿泊ゲストが LINE からアクセスした場合は、設定項目は一切表示せず**、カレンダー予約画面などゲスト向けの画面のみ表示する。
* **管理者向け:** 管理者の LINE でアクセスした時のみ、設定編集・月次PDF用 Drive フォルダID・月締め実行等の画面が表示される。主にPC・タブレットでの利用を想定。
* **ゲスト向け:** カレンダー予約・空き確認・予約確定のみ。設定画面へのリンクやメニューは表示しない。
* **一貫性:** ボタン・保存状態・エラー表示を明確にし、編集内容が確実にスプレッドシートに反映されることを示す。

## 3. 管理者画面（Vercel）詳細
**表示条件:** LIFF で取得した LINE User ID が config の `adminLineUserId` と一致する場合のみ表示する。一致しない（ゲスト）場合は、このセクションの画面は表示せず、カレンダー予約画面のみ表示する。

### 3.1 レイアウト方針
* **背景:** 薄いグレー（#f0f2f5）など、家計簿アプリと同様のトーンで統一可能。
* **コンテナ:** 中央寄せ、角丸、白背景。最大幅 600px 程度で可読性を確保。
* **管理者メニュー:** トップまたはサイドに「設定」へのリンクを配置。各設定項目はタブまたはアコーディオンで切り替えてもよい。

### 3.2 管理者のみ編集可能な設定項目一覧
以下の項目は、**管理者の LINE でアクセスした時のみ** Vercel 画面上で表示・編集できる。宿泊ゲストがアクセスした場合は**見えなくてよい**（表示しない）。

| 設定項目 | 内容・保存先 | 備考 |
| :--- | :--- | :--- |
| **Google Drive フォルダID** | 月間売上 PDF の保存先。config: `driveFolderId` | 管理者が Drive でフォルダを作成し、URL からフォルダID を取得して入力 |
| **返信メッセージの編集** | 問い合わせ等への返信用テンプレート。templates または config | |
| **予約確定メッセージ** | ゲストに送る予約確定時の御礼文面。templates: `reservationConfirm` 等 | |
| **一泊単価（価格設定）** | config: `nightlyRate` | 数値（円） |
| **清掃代（価格設定）** | config: `cleaningFee` | 数値（円） |
| **ペット宿泊ありの場合の清掃代** | config: `cleaningFeeWithPet` | 数値（円） |
| **周辺のおすすめレストラン案内** | 予約確定時にゲストに送る文章。templates: `restaurantGuide` 等 | |
| **チェックインガイドPDFのURL** | チェックイン3日前に送るURL。config: `checkInGuidePdfUrl` | |
| **宿泊1週間前の案内メッセージ** | templates: `oneWeekBefore` | 既存 |
| **玄関ドアの開け方動画のURL** | チェックイン当日お昼に送るURL。config: `doorVideoYoutubeUrl` | |

各項目はテキスト入力または数値入力とし、保存ボタンで GAS の API（saveConfig / saveTemplate）を呼び出して config または templates に保存する。

### 3.3 月次売上 PDF 用 Drive フォルダID 設定
* **入力欄:** 1行テキスト。Google Drive のフォルダを開いたときの URL の `folders/` 以降の ID を入力する（例: `1RkQ9GF9teR76ufQ4lUvra2Nx90S3cR-Q`）。
* **保存:** `driveFolderId` として config に保存。管理者以外はこの入力欄自体を表示しない。

### 3.4 月締め・PDF 出力
* **トリガー方法（いずれかまたは両方）:**  
  (1) 管理者画面に「○年○月を締める」ボタンを配置し、押下時に GAS の「月次集計・PDF生成」API を呼び出す。  
  (2) 時間駆動で毎月1日に前月を自動締めし、PDF を生成して Drive に保存する。
* **表示:** 管理者画面に「月締め実行」または「前月の売上PDFを生成」のようなボタン／実行履歴を表示してもよい。

### 3.5 通知テンプレート編集（既存の「1週間前」等）
* 上記 3.2 の一覧に含まれる「宿泊1週間前の案内メッセージ」「予約確定メッセージ」等は、同じく管理者のみ表示の編集エリアで `<textarea>` 等により編集する。
* 保存時に GAS の saveTemplate / saveConfig を呼び出し、成功時は「保存しました」を表示する。

### 3.6 今後の拡張（案）
* 予約一覧の参照、清掃員グループID の設定画面（管理者のみ）を追加する。

## 3.7 カレンダー予約画面（Vercel・ゲスト向け）

ゲストが LINE から LIFF で開く、または同一 Vercel サイトの予約ページ。Airbnb iCal を元に空き状況を表示し、チェックイン・チェックアウト・宿泊人数を選択して総支払金額を表示し、確定ボタンで予約を完了する。

### A. 認証
* **LINE Login（LIFF）:** ページ表示時に LIFF を初期化し、未ログインの場合は `liff.login()` でログインさせる。ログイン後に `liff.getProfile()` で `userId`（LINE User ID）と `displayName` を取得し、予約確定時にこのユーザーに紐付ける。
* 未ログインのままでは「宿泊を確定させる」を押せない、または押した時点でログインを促す。

### B. カレンダー表示
* **形式:** 月表示のカレンダー（例: FullCalendar 等のライブラリ、または独自実装）。対象月は数ヶ月先まで選択可能とする。
* **空き・予約済みの表示:** GAS の `getAvailability` API で取得した「予約済み（ブロック）日」一覧に基づき、予約済みの日は選択不可またはグレーアウト、空きの日は選択可能として表示する。
* **チェックイン日選択:** 空きの日をタップ/クリックでチェックイン日として選択。選択後、チェックアウト候補を絞り込む（チェックイン以降で、かつ連続した空き日のみ等のルールを適用可能）。

### C. チェックアウト日・宿泊人数
* **チェックアウト日:** カレンダーまたは別の日付ピッカーで、チェックイン日より後の空き日から選択。最低1泊（チェックアウト = チェックインの翌日以降）とする。
* **宿泊人数:** プルダウン（例: 1人〜10人）または数値入力。料金が人数で変わる場合は config のルールに従い、変わらない場合は記録用のみ。

### D. 総支払金額の表示
* **計算式:** 宿泊料金（1泊単価 × 泊数）＋ 清掃代（10,000円）。1泊単価は GAS の config または API で取得する。
* **表示:** 「総支払金額: ¥○○,○○○」のように大きく表示。内訳（宿泊料金・清掃代）を小さく表示してもよい。

### E. 宿泊を確定させるボタン
* **ラベル:** 「宿泊を確定させる」「予約を確定する」など。
* **動作:** クリック時に、選択内容（チェックイン日・チェックアウト日・宿泊人数・計算した金額）を GAS の `confirmReservation` API に送信。成功時は「予約が確定しました」のメッセージを表示し、ゲストの LINE に御礼メッセージ、管理者の LINE に予約着信通知が送られる。失敗時（例: その期間が既に予約済み）はエラーメッセージを表示する。
* **二重送信防止:** 送信中はボタンを非活性化する。

### F. 状態表示・エラー
* 予約確定成功: 「予約が確定しました。LINEで御礼メッセージをお送りしています。」等を表示。
* 予約確定失敗: 「申し訳ありません。その期間は既に予約が入っている可能性があります。」等を表示し、再度日付を選び直せるようにする。

## 4. ゲスト体験（LINEメッセージ）の流れ

### 4.1 問い合わせ〜予約希望
* ゲストがLINEでテキストを送信。
* システムが受信し、スプレッドシートに保存。管理者に通知。
* （運用で）管理者が返信し、空きがあれば「宿泊料金＋清掃代1万円」「ご予約確定しますか？」を送る。

### 4.2 予約確定後
* ゲストに「予約確定」＋おすすめレストラン案内が送信される。
* 以降、以下のタイミングで自動メッセージが届く:
  - チェックイン3日前: チェックインガイドPDF URL
  - チェックイン1週間前: 編集可能な「1週間前」案内文
  - チェックイン当日お昼: 玄関ドア開け方（YouTube URL）
  - チェックイン当日16時頃: チェックイン完了確認・補足
  - チェックアウト前日お昼: 明日チェックアウト・お忘れ物・清掃協力の案内

### 4.3 メッセージのトーン
* 丁寧で分かりやすい短文。必要に応じて改行や絵文字を控えめに使用する。

## 4. 写真ギャラリー画面（Vercel）

### 4.1 表示条件
* **全ユーザー:** 写真ギャラリーは全ユーザーが閲覧可能。
* **管理者のみ:** 写真のアップロード・並び替え・削除機能は管理者のみ表示。

### 4.2 ゲスト向け表示
* **写真ギャラリー:** グリッド表示で施設写真を一覧表示。
* **クリック:** 写真をクリックすると拡大表示（新しいタブで開く）。
* **説明文:** 各写真の下に説明文（caption）を表示。

### 4.3 管理者向け機能
* **写真を選択:** `<input type="file" accept="image/*">` でスマホから写真を選択。
* **説明文（オプション）:** 写真の説明文を入力（例: リビングルーム）。
* **写真をアップロード:** クリックで写真をBase64エンコードしてGASに送信。GASはGoogle Driveに保存し、サムネイルURLを返す。
* **並び替え:** ドラッグ&ドロップで写真の表示順を変更。
* **削除:** 各写真に「削除」ボタンを配置。クリックで写真をリストから削除。
* **並び順を保存:** 並び替え後、このボタンをクリックして変更をスプレッドシートに保存。

### 4.4 技術仕様
* **画像形式:** JPEG, PNG, GIF, WebP, BMP, SVG, HEIC/HEIF（スマホ写真）対応。
* **ストレージ:** Google Driveの指定フォルダ（`photosFolderId`）に保存。
* **URL形式:** `https://drive.google.com/thumbnail?id=FILEID&sz=w1000`（サムネイル表示）。
* **データ管理:** スプレッドシートの`photos`シート（photoId, imageUrl, displayOrder, caption）。

## 6. インタラクション・状態遷移（管理者画面）

1. **初期表示:** ページ読み込み後、GASの `getTemplate` 等で「1週間前」の現在の文面を取得し、`textarea` に表示する。
2. **編集:** ユーザーがテキストを編集。
3. **保存:** 「保存する」クリック → ボタン非活性 → GASへPOST → 成功時「保存しました」表示・ボタン復帰。失敗時はエラー表示・ボタン復帰。
4. **再読み込み:** 必要に応じて「再読み込み」で最新テンプレートを取得して表示する。
