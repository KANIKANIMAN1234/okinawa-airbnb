# Airbnb iCal・カレンダー予約仕様書 (v1.0)

## 1. 概要
本ドキュメントは、Airbnb の iCal カレンダーURL を連携し、Vercel 上の Web サイトに空き状況をカレンダー形式で表示。ゲストがチェックイン日・チェックアウト日・宿泊人数を選択し、総支払金額を確認したうえで「宿泊を確定させる」により予約を完了する一連の仕様をまとめる。予約確定時にはゲスト・管理者の LINE に自動通知し、Airbnb カレンダーとの同期方針を定義する。

---

## 2. Airbnb iCal 連携

### 2.1 提供URL（例）
* **iCal URL（読み取り用）:**  
  `https://www.airbnb.jp/calendar/ical/20183082.ics?t=cd7bda003b594bda893633333e18460e`
* 上記は Airbnb の「予約カレンダーをエクスポート」で取得できる形式。**他のカレンダーと予約情報を共有する**ためのエクスポート用URLである。

### 2.2 役割と制約
* **読み取り（実装する）:** 当システムの GAS がこの URL にアクセスし、iCal（.ics）形式のデータを取得。VEVENT をパースして「予約済み（ブロック）日」を抽出し、空き状況の元データとして利用する。
* **書き込み（現状は不可）:** Airbnb の iCal は**エクスポート（読み取り）専用**。当システムからこの URL 経由で Airbnb に予約やブロックを書き込むことはできない。
* **Airbnb 側への反映:** 当システムで予約が確定した場合、管理者が Airbnb の管理画面（カレンダー・予約設定）で該当期間を**手動でブロック**する運用とする。将来、Airbnb がホスト向けにカレンダー更新 API を提供した場合は、その API で自動反映する拡張を検討する。

### 2.3 データの更新遅延
* Airbnb の iCal は最大で数十分遅れて更新される場合がある。そのため、予約確定時には再度 iCal を取得して空きを確認し、競合がなければ保存する。二重予約の防止は「確定直前の再チェック」と「管理者による Airbnb 手動ブロック」で担保する。

---

## 3. 空き状況の取得・表示

### 3.1 取得フロー
1. Vercel のカレンダー画面で、表示対象の期間（例: 年・月）を指定する。
2. フロントから GAS の `getAvailability` API を呼び出す（CORS を避けるため、.ics は GAS 経由でのみ取得する）。
3. GAS は config の `airbnbIcalUrl` を使って iCal を取得し、VEVENT をパースして「予約済み日」のリスト（blockedDates: YYYY-MM-DD の配列）を返す。
4. フロントでは、表示対象の全日付のうち blockedDates に含まれない日を「空き」、含まれる日を「予約済み」としてカレンダー上に表示する。

### 3.2 カレンダー表示仕様
* **形式:** 月単位のカレンダー（例: FullCalendar 等のライブラリまたは独自実装）。数ヶ月先まで表示・選択可能とする。
* **空き:** 選択可能。チェックイン・チェックアウトの候補とする。
* **予約済み:** 選択不可、またはグレーアウト等で視覚的に区別する。

## 4. 予約条件の選択と金額表示

### 4.1 選択項目
| 項目 | 内容 |
| :--- | :--- |
| **チェックイン日** | カレンダー上で空きの日から選択。 |
| **チェックアウト日** | チェックイン日より後で、空きの日から選択。最低1泊（チェックアウト ≥ チェックイン翌日）。 |
| **宿泊人数** | プルダウンまたは数値で選択（例: 1〜10人）。予約レコードに保存し、必要に応じて料金ルールに反映する。 |

### 4.2 総支払金額の計算と表示
* **計算式:** 総支払金額 = 宿泊料金（1泊単価 × 泊数）＋ 清掃代（10,000円）
* **1泊単価:** config の `nightlyRate` で管理。GAS の `getPricing` API でフロントに返す。
* **泊数:** チェックアウト日 − チェックイン日（日数）。例: 5/1 チェックイン・5/3 チェックアウト → 2泊。
* **表示:** 画面上に「総支払金額: ¥○○,○○○」と表示。内訳（宿泊料金・清掃代）を併記してもよい。

## 5. 宿泊確定フロー

### 5.1 前提
* ゲストは **LINE Login（LIFF）** でカレンダー画面にアクセスしている。これにより、確定時に予約者と紐づける **LINE User ID** を取得できる。

### 5.2 「宿泊を確定させる」ボタン
* 押下時に、選択内容（チェックイン日・チェックアウト日・宿泊人数・総支払金額）と、LIFF で取得した lineUserId・displayName を GAS の `confirmReservation` API に送信する。

### 5.3 GAS 側の処理（confirmReservation）
1. 受信データでチェックイン〜チェックアウトの期間を特定する。
2. 再度 `airbnbIcalUrl` から iCal を取得し、その期間がブロック日と重複していないか確認する。
3. **重複あり:** `status: "error"`, `message: "選択された期間は既に予約が入っています"` を返す。フロントでエラー表示する。
4. **重複なし:**  
   - reservations シートに新規行を追加（ゲストLINE User ID, チェックイン, チェックアウト, 宿泊人数, 金額, 清掃代, ステータス「確定」, 確定日時）。  
   - guests シートに未登録ならゲストを追加。  
   - **ゲストの LINE** に御礼メッセージ（予約確定の旨・日程・金額等）を push。  
   - **管理者の LINE** に「予約が入りました」と、ゲスト名・日程・人数・金額の要約を push。  
   - `status: "success"`, `result: { reservationId }` を返す。

### 5.4 確定後の案内
* 既存仕様と同様、予約確定後に「周辺おすすめレストラン」「チェックイン3日前のガイド」「1週間前メッセージ」等の自動通知が、時間駆動トリガーによりゲストの LINE に送信される。

## 6. Airbnb カレンダーとの「同期」の整理

| 項目 | 内容 |
| :--- | :--- |
| **当システム → Airbnb（書き込み）** | iCal URL では不可。**管理者が Airbnb 管理画面で該当期間を手動ブロック**する運用。確定通知に「Airbnb で該当日程をブロックしてください」等の案内を含めてもよい。 |
| **Airbnb → 当システム（読み取り）** | iCal URL で取得し、空き状況としてカレンダーに反映。**実装する。** |
| **将来** | Airbnb がホスト向け API でカレンダー更新を提供した場合、確定時にその API を呼び出してブロックを追加する拡張を検討する。 |

## 7. config に登録する項目（参照）

| configKey | 説明 | 例 |
| :--- | :--- | :--- |
| **airbnbIcalUrl** | Airbnb の iCal エクスポートURL | `https://www.airbnb.jp/calendar/ical/20183082.ics?t=...` |
| **nightlyRate** | 1泊あたりの宿泊料金（円） | `15000` |
| **cleaningFee** | 清掃代（円） | `10000` |

---

## 8. まとめ
* **できること:** Airbnb の iCal を読み取り、Vercel 上に空き状況カレンダーを表示。チェックイン・チェックアウト・宿泊人数を選択し、総支払金額を表示したうえで「宿泊を確定させる」で予約完了。ゲスト・管理者の LINE に自動通知。
* **Airbnb への反映:** iCal は読み取り専用のため、確定予約を Airbnb カレンダーに反映するには**管理者による手動ブロック**を行う。当システムは確定内容をスプレッドシートに保存し、管理者通知で案内することで、二重予約を防ぐ運用をサポートする。
