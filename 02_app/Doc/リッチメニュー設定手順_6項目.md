# リッチメニュー設定手順（6項目）

以下の6項目でリッチメニューを設定するときの手順です。

| # | メニュー名 | 種別 | 内容 |
|---|------------|------|------|
| 1 | 空き情報・予約する | リンク | 予約画面のURLを開く |
| 2 | 施設情報 | テキスト or ポストバック | 施設情報の案内を返信 |
| 3 | 周辺・レストラン | テキスト or ポストバック | 周辺・レストラン案内を返信 |
| 4 | 利用規約 | テキスト or ポストバック | 利用規約の案内を返信 |
| 5 | メッセージで問い合わせ | テキスト or ポストバック | 「メッセージでお送りください」と返信 |
| 6 | アクセスについて | テキスト or ポストバック | アクセス案内を返信 |

---

## 重要：「ポストバック」はどこにあるか

**LINE 公式アカウントマネージャー**（manager.line.biz などでリッチメニューを編集する画面）では、  
アクションの「タイプ」に **リンク・テキスト・クーポン・ショップカード・設定しない** しかなく、**「ポストバック」は出てきません**。  
これは仕様で、ポストバックは **Messaging API 用のリッチメニュー** でしか使えません。

そのため、次の **2つのやり方** のどちらかで設定します。

| 方法 | どこで作るか | 2〜6番の設定 |
|------|--------------|----------------|
| **方法A** | **LINE Developers Console**（developers.line.biz）で Messaging API からリッチメニューを作成 | タイプ「**ポストバック**」、データに `facility_info` などを入力 |
| **方法B** | **LINE 公式アカウントマネージャー**（今の画面）でリッチメニューを作成 | タイプ「**テキスト**」、送信するテキストに `facility_info` などを入力 |

- **今、マネージャーで「タイプ」を選んでいる画面なら → 下の「方法B」に進んでください。**
- ポストバックを使いたい場合は、**方法A**（Developers Console で作成）に切り替える必要があります。

---

## 方法B：公式アカウントマネージャーで「テキスト」を使う（おすすめ・素人向け）

マネージャーには「ポストバック」がないので、**「テキスト」** で代用します。

- ユーザーがメニューをタップすると、**指定した文字列がトークに送信**されます（ユーザーが自分で打ったように扱われます）。
- その文字列を **GAS が受け取り**、施設情報やアクセス案内などの**返信文を自動で返す**ようにしてあります。
- 入力する文字列は **ユーザーに意味が分からなくても問題ありません**（`facility_info` などでOK）。  
  トークには一瞬その文字が表示されますが、すぐにこちらの案内文が返ります。

### 1番「空き情報・予約する」の設定（マネージャー）

| 項目 | 操作 |
|------|------|
| **タイプ** | **リンク** を選ぶ |
| **URL（リンク先）** | 予約画面のURLを入力  
  例: `https://okinawa-airbnb.vercel.app` |
| **リンクの開き方** | 「LINE内ブラウザで開く」など、そのままでOK |

※ 項目名は「リンク」「URL」「リンク先」など、画面によって少し違う場合があります。

---

### 2〜6番の設定（マネージャーで「テキスト」）

2番〜6番は、**タイプで「テキスト」を選び**、**「送信するテキスト」**（または「テキスト」「メッセージ」などと書かれた入力欄）に、**下の表の文字をそのまま**入力します。

| メニュー | タイプ | 送信するテキスト（ここにコピー） |
|----------|--------|----------------------------------|
| 2. 施設情報 | **テキスト** | `facility_info` |
| 3. 周辺・レストラン | **テキスト** | `restaurant_guide` |
| 4. 利用規約 | **テキスト** | `terms_of_use` |
| 5. メッセージで問い合わせ | **テキスト** | `menu_contact` |
| 6. アクセスについて | **テキスト** | `access_info` |

**操作の流れ（1つずつ）：**

1. エリア（A, B, C…）を1つ選ぶ。
2. **「タイプ」** のプルダウンを開く。
3. **「テキスト」** を選ぶ。
4. 「テキスト」を選ぶと、**送信するメッセージ**や**テキスト**を入力する欄が出ます。  
   そこに、上表の **英小文字の文字列** を **そのまま** 入力（またはコピー＆ペースト）。  
   例：2番なら `facility_info` だけ。スペースや改行は入れない。
5. 保存して、次のエリアも同様に設定。

**注意：**

- 入力するのは **半角英小文字** だけです。`facility_info` のように、アンダースコア（_）も含めます。
- ユーザーがタップすると、この文字がトークに送信され、GAS が受け取って案内文を返します。  
  GAS 側の対応は済んでいます（`LineWebhook.gs` で「テキスト」で届いた場合も同じ案内を返すようにしています）。

---

## 方法A：LINE Developers Console でポストバックを使う場合

ポストバックを使う場合は、**LINE Developers Console**（https://developers.line.biz/console/）で、**Messaging API** 用のリッチメニューを作成します。  
ここでは「タイプ」に **ポストバック** が選べ、**データ** に `facility_info` などを入れます。  
手順の詳細は、このドキュメントの後半「ステップ2：LINE Developers でリッチメニューを作成する」以降を参照してください。

※ マネージャーと Developers の**両方**にリッチメニューがある場合、表示の優先度が決まっています。  
  **マネージャーで作成したリッチメニューを優先したい**ときは、方法B（テキスト）でそちらを設定してください。

---

## 前提

- LINE Developers でチャネル（Messaging API）を作成済みであること
- GAS の Webhook がデプロイ済みで、LINE の Webhook URL に設定済みであること
- 予約画面（LIFF）の URL が決まっていること（例: `https://okinawa-airbnb.vercel.app`）

---

## ステップ1：リッチメニュー用の画像を用意する

### サイズ

- **推奨:** **2500×1686 px**（2行×3列で6エリア）
- または **2500×843 px**（1行×6列）

### レイアウト例（2行×3列の場合）

```
[ 1. 空き情報・予約する ] [ 2. 施設情報     ] [ 3. 周辺・レストラン ]
[ 4. 利用規約           ] [ 5. メッセージ問合せ ] [ 6. アクセスについて ]
```

- 画像は **PNG または JPEG**
- 各ブロックに「1」「2」…などのラベルやアイコンを描くと、ユーザーが押しやすいです
- ファイルサイズは **1MB 以下**（推奨 500KB 以下）

---

## ステップ2：LINE Developers でリッチメニューを作成する

1. **LINE Developers Console** を開く  
   https://developers.line.biz/console/

2. 対象の **プロバイダー** → **チャネル**（Messaging API）を選択

3. 左メニューで **「リッチメニュー」** をクリック

4. **「作成」** ボタンをクリック

5. **基本設定** を入力  
   - **リッチメニュー名:** 例）`沖縄Airbnb メインメニュー`
   - **チャットバー テキスト:** 例）`メニューをタップしてください`
   - **サイズ:** **2500×1686**（2行3列）または **2500×843**（1行6列）を選択
   - **デフォルト表示:** オン推奨（友だち追加時にこのメニューを表示）

6. **「次のステップ」** をクリック

## ステップ3：エリア（タップ領域）を設定する

画像のどの範囲をタップするかを、**エリア**で設定します。

### 2行×3列（2500×1686）の場合

1行あたりの高さ: 1686÷2 = **843 px**  
1列あたりの幅: 2500÷3 ≒ **833 px**

| メニュー | エリア名（メモ用） | X | Y | 幅 width | 高さ height | アクション種別 | 設定値 |
|----------|-------------------|---|---|----------|-------------|----------------|--------|
| 1. 空き情報・予約する | 予約 | 0 | 0 | 833 | 843 | **URI** | 下記参照 |
| 2. 施設情報 | 施設 | 833 | 0 | 834 | 843 | **postback** | `facility_info` |
| 3. 周辺・レストラン | レストラン | 1666 | 0 | 834 | 843 | **postback** | `restaurant_guide` |
| 4. 利用規約 | 規約 | 0 | 843 | 833 | 843 | **postback** | `terms_of_use` |
| 5. メッセージで問い合わせ | 問合せ | 833 | 843 | 834 | 843 | **postback** | `menu_contact` |
| 6. アクセスについて | アクセス | 1666 | 843 | 834 | 843 | **postback** | `access_info` |

### 1の「URI」の設定値

- **アクション:** URI
- **URI:** 予約画面の URL  
  - 例: `https://okinawa-airbnb.vercel.app`  
  - LIFF を使う場合は、LIFF のエンドポイント URL をそのまま入力
- **ラベル:** 空欄でも可。入力する場合は「予約する」など短く

## 【詳解】2〜6番の「ポストバック」の設定のしかた（方法A・Developers Console 用）

**※ これは LINE Developers Console でリッチメニューを作る場合の説明です。**  
公式アカウントマネージャーで設定する場合は、上記「方法B：テキストを使う」に従ってください。

ここでは、**2番〜6番の各エリア**で「ポストバック」を選び、**データ**に何を入力するかを、画面の流れで説明します。

---

### ポストバックとは（イメージ）

- **URI** … タップすると「ブラウザで指定したURLを開く」
- **ポストバック** … タップすると「LINEがサーバー（GAS）に『このボタンが押されました』という情報を送る」だけ。  
  サーバー側で「どのボタンか」を区別するために、**データ**に**決まった文字列**を入れます。  
  この文字列を GAS が受け取り、対応する案内文を返信します。

---

### 画面の流れ（共通）

リッチメニュー編集画面で、**エリアを1つ追加したあと**、だいたい次のような項目が並びます。

1. **エリアの位置・大きさ**  
   - 「X」「Y」「幅」「高さ」の数値（ステップ3の表のとおりに入力）

2. **アクション（動作の種類）**  
   - ここで **「ポストバック」** を選びます。  
   - 選択肢は「メッセージ」「URI」「ポストバック」など。  
     → **「ポストバック」** を選んでください。

3. **「ポストバック」を選ぶと出てくる欄**  
   - **データ**（または「Data」「ポストバックデータ」など）  
     → ここに **下の「入力する文字」** を**そのまま**コピーして貼り付けます。  
   - **ラベル**（あれば）  
     → 空欄でかまいません。入れる場合は「施設情報」など短く。

4. **保存**  
   - そのエリアを保存し、次のエリアを同じ手順で追加していきます。

### 操作の順番のおさらい（2〜6番）

1. 2番のエリアを追加 → 位置・サイズを入力 → アクションで「**ポストバック**」→ データに「**facility_info**」→ 保存  
2. 3番のエリアを追加 → 同様に「**ポストバック**」→ データに「**restaurant_guide**」→ 保存  
3. 4番のエリアを追加 → 「**ポストバック**」→ データに「**terms_of_use**」→ 保存  
4. 5番のエリアを追加 → 「**ポストバック**」→ データに「**menu_contact**」→ 保存  
5. 6番のエリアを追加 → 「**ポストバック**」→ データに「**access_info**」→ 保存  

これで、ユーザーがリッチメニューの2〜6をタップしたときに、GAS がそれぞれの「データ」を受け取り、対応する案内文を返信できるようになります。

### 1行×6列（2500×843）の場合

高さは 843 px のまま。幅を 6 等分: 2500÷6 ≒ **416 px**（端数は 417 などで調整）

| メニュー | X | Y | 幅 | 高さ | アクション | データ／URI |
|----------|---|---|-----|------|------------|-------------|
| 1 | 0 | 0 | 417 | 843 | URI | 予約画面URL |
| 2 | 417 | 0 | 417 | 843 | postback | facility_info |
| 3 | 834 | 0 | 416 | 843 | postback | restaurant_guide |
| 4 | 1250 | 0 | 417 | 843 | postback | terms_of_use |
| 5 | 1667 | 0 | 416 | 843 | postback | menu_contact |
| 6 | 2083 | 0 | 417 | 843 | postback | access_info |

## ステップ4：エリアを LINE の画面で登録する

1. リッチメニュー編集画面で **「エリアの追加」** または **「エリアを設定」** をクリック

2. 上表のとおり、**6回** エリアを追加する  
   - 1回目: X=0, Y=0, 幅=833, 高さ=843 → アクション「URI」、URI に予約画面 URL  
   - 2回目: X=833, Y=0, 幅=834, 高さ=843 → アクション「ポストバック」、データ `facility_info`  
   - 3回目: X=1666, Y=0, 幅=834, 高さ=843 → ポストバック `restaurant_guide`  
   - 4回目: X=0, Y=843, 幅=833, 高さ=843 → ポストバック `terms_of_use`  
   - 5回目: X=833, Y=843, 幅=834, 高さ=843 → ポストバック `menu_contact`  
   - 6回目: X=1666, Y=843, 幅=834, 高さ=843 → ポストバック `access_info`

3. **画像をアップロード**（ステップ1で用意した画像）

4. **「公開」** または **「作成」** で保存

5. **「デフォルト」に設定** すると、友だち追加時にこのリッチメニューが表示されます

## ステップ5：GAS 側の postback 対応（実装済みの場合）

GAS の `LineWebhook.gs` で、次の postback データを受け取り、テンプレート文面を返信する処理が入っています。

| postback data | テンプレートキー（templates シート） | 説明 |
|---------------|-------------------------------------|------|
| `facility_info` | facilityInfo | 施設情報 |
| `restaurant_guide` | restaurantGuide | 周辺・レストラン |
| `terms_of_use` | termsOfUse | 利用規約 |
| `menu_contact` | contactPrompt | メッセージで問い合わせの誘導文 |
| `access_info` | accessInfo | アクセスについて |

**やること:**  
Google スプレッドシートの **templates** シートに、上記の **templateKey** の行を追加し、**本文（body）** に表示したい文面を入力してください。  
Vercel の管理者画面で「メッセージテンプレート」を増やしていれば、同じキーで編集・保存できます。

## ステップ6：テンプレートの初期文面をスプレッドシートに追加する

templates シートに以下の行が無い場合は追加します。

| テンプレートキー | 本文（例） |
|-----------------|------------|
| facilityInfo | 施設のご案内です。○部屋、○人まで宿泊可能です。Wi-Fi・洗濯機・キッチン完備。詳細はメッセージでお問い合わせください。 |
| restaurantGuide | （既存の restaurantGuide をそのまま利用）周辺のオススメレストランをご案内します。… |
| termsOfUse | 利用規約はこちらです。[URLや条文を記載]。ご予約をもって規約に同意いただいたものとみなします。 |
| contactPrompt | ご不明点は、このチャットにメッセージでお送りください。担当者よりご返信いたします。 |
| accessInfo | アクセスのご案内です。[最寄り駅・バス停・車での行き方・駐車場などを記載]。 |

※ 本文は後から Vercel の管理者画面で編集できます。

## うまく動かないとき

- **タップしても何も返ってこない**  
  - Webhook URL が GAS のデプロイ URL になっているか確認  
  - GAS の「実行ログ」で postback が届いているか確認  
  - templates に該当の templateKey があるか、body が空でないか確認  

- **「1. 空き情報・予約する」で画面が開かない**  
  - URI に入力した URL が正しいか（https:// から始まるか）確認  
  - LIFF のエンドポイント URL と一致しているか確認  

- **表示されるリッチメニューが古い**  
  - キャッシュのため数分かかることがあります。LINE アプリを再起動するか、しばらく待ってから再度表示を確認してください。
