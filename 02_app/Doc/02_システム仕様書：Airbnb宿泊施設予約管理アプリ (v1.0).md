# システム仕様書：Airbnb宿泊施設予約管理アプリ (v1.0)

## 1. システム概要
本システムは、LINE公式アカウントを窓口とした宿泊施設の予約管理ソリューションである。ゲストはLINEで問い合わせ・予約希望を送信し、管理者（井上）はWebhookでその内容を即時に把握する。空き状況に応じて宿泊料金・清掃代（1万円）の案内および予約確定を行い、確定後はチェックイン前後の各種案内（1週間前・3日前・当日・チェックアウト前日）を自動通知する。Vercel上の管理者画面で「1週間前」メッセージを編集でき、その内容はGoogleスプレッドシートに保存され自動通知に反映される。

---

## 2. システムアーキテクチャ
3層構造のサーバーレス構成を採用する。

### 2.1 フロントエンド（Presentation Layer）
- **ホスティング:** Vercel（GitHub連携）
- **技術スタック:** HTML5, CSS3, JavaScript (ES6+)、必要に応じてカレンダーUIライブラリ（例: FullCalendar 等）
- **役割:**
  - **管理者用画面（管理者の LINE ID でアクセスした時のみ表示）:** 返信メッセージ・予約確定メッセージ・一泊単価・清掃代・ペット時清掃代・おすすめレストラン案内・チェックインガイドPDF URL・宿泊1週間前メッセージ・玄関ドア動画URL・**月次PDF用 Google Drive フォルダID**・**施設写真の管理（アップロード・並び替え・削除）** の編集。編集内容は GAS 経由で config / templates / photos に保存。
  - **ゲスト向け画面:** 宿泊ゲストが LINE から LIFF でアクセスした場合は**設定項目は一切表示せず**、カレンダー予約画面と**施設写真ギャラリー**のみ表示する。LIFF で取得した LINE User ID が config の管理者 ID と一致する場合のみ管理者用メニュー・設定画面・写真管理機能を表示する。
  - **ゲスト向けカレンダー画面:** Airbnb iCal を元にした空き状況のカレンダー表示、チェックイン・チェックアウト・宿泊人数の選択、総支払金額の表示、宿泊確定ボタン。LINE Login（LIFF）でゲストを識別し、確定時に予約者に紐付ける。
  - **写真ギャラリー:** 全ユーザーが閲覧可能な施設写真ギャラリー。管理者のみ写真のアップロード・並び替え・削除が可能。

### 2.2 バックエンド（Application Layer）
- **プラットフォーム:** Google Apps Script (GAS)
- **役割:**
  - LINE Webhook の受信・署名検証・メッセージのパース。
  - 問い合わせ・予約データのスプレッドシートへの保存・取得。
  - 管理者LINEへの通知送信（LINE Messaging API）。
  - ゲストへのプッシュメッセージ送信（予約確定通知、1週間前・3日前・当日・チェックアウト前日の案内）。
  - 時間駆動トリガーによる定期実行（通知タイミングの制御）。
  - Vercel管理者画面からのテンプレート保存APIの提供。
  - **Airbnb iCal の取得・パース:** 登録した iCal URL を定期的またはリクエスト時に取得し、予約済み日（ブロック日）を抽出。空き日一覧をAPIでフロントに返す。
  - **カレンダー予約API:** チェックイン・チェックアウト・人数・金額を受け取り、空き確認後に予約を保存。ゲスト・管理者・**清掃員用 LINE グループ／オープンチャット**へLINE通知。必要に応じて予約データを .ics 形式でエクスポートする機能（管理者が手動でAirbnbに反映する際の補助）。
  - **月間売上集計・PDF保存:** 月締め時に、対象月の確定予約を集計し、PDF を生成して config で指定した Google Drive フォルダに保存する。フォルダID は管理者画面でのみ設定可能。
  - **管理者判定API:** Vercel から LIFF で取得した LINE User ID を送信し、管理者かどうかを返す。管理者の場合のみ設定画面の表示を許可する。
  - **写真管理API:** 
    - **写真アップロード:** 管理者がスマホから選択した写真をBase64エンコードしてGASに送信。GASはGoogle Driveの指定フォルダに保存し、サムネイルURLを返す。
    - **写真一覧取得:** スプレッドシートの`photos`シートから写真一覧（URL・表示順・説明文）を取得。
    - **写真並び替え・保存:** ドラッグ&ドロップで並び替えた順序をスプレッドシートに保存。
    - **写真削除:** 指定した写真をリストから削除（Google Drive上のファイルは残る）。

### 2.3 データ・ストレージ（Data Layer）
- **データベース:** Google Sheets（スプレッドシート）
  - 問い合わせ履歴、予約一覧、ゲスト（LINE User ID）マスタ、通知テンプレート文面、管理者LINE User ID 等を保存。
- **認証・連携:** LINE Login（ゲスト識別）、LINE Messaging API（送受信）、LINE Webhook（イベント受信）。

## 3. 機能詳細仕様

### 3.1 権限・ユーザー識別
- **ゲスト:** LINE公式アカウントでメッセージを送ったユーザーの `userId`（LINE User ID）で識別。問い合わせ・予約はすべてこのIDに紐付ける。
- **管理者:** あらかじめ登録した **管理者LINE User ID**（井上）にのみ、問い合わせ着信通知・予約確定通知を送信する。
- **判定:** Webhookで受け取った `userId` が管理者IDと一致する場合は管理者向け処理、一致しない場合はゲストとして問い合わせ・予約フローを扱う。

### 3.2 問い合わせ受付（Webhook）
1. LINE Platform から Webhook でメッセージイベントがGASに送信される。
2. 署名検証によりリクエストの正当性を確認する。
3. 送信者の `userId` とメッセージ本文・タイムスタンプを取得。
4. スプレッドシートの「問い合わせ」シートに、LINE User ID・メッセージ内容・受信日時を1行で保存する。
5. 送信者がゲストの場合、管理者のLINE User ID 宛に「ゲストからの問い合わせ」および内容の要約（または本文）をプッシュ通知する。

### 3.3 予約希望〜確定フロー
- **予約希望:** ゲストのメッセージを「予約希望」として扱う場合（キーワードやフォーマットは仕様で定義）、スプレッドシートの「予約」関連シートに希望日等を記録する。
- **空き確認:** 管理者がカレンダー等で空きを確認し、空きの場合にゲストへ「宿泊料金＋清掃代1万円」「ご予約確定しますか？」を返信する（手動または半自動を想定）。
- **予約確定:** ゲストが確定の意思表示をした場合、予約レコードを「確定」に更新する。
- **確定時通知:**
  - ゲスト: 予約確定のメッセージ（config/templates で編集可能）＋周辺おすすめレストラン案内を送信。
  - 管理者: 管理者LINEに「予約確定（ゲスト名・日程・人数・金額等）」を通知。
  - 清掃員: config に登録した **清掃員用 LINE グループID（またはオープンチャットのグループID）** 宛に、予約確定の通知（チェックイン・チェックアウト日等）をプッシュ送信する。ボットをそのグループ／オープンチャットに追加し、取得したグループIDを config（例: cleanerLineGroupId）に保存する。

### 3.4 自動通知（ゲスト向け）のタイミングと内容
GASの時間駆動トリガーで、予約データのチェックイン・チェックアウト日を参照し、以下のタイミングでゲストのLINEにメッセージを送信する。

| トリガー条件 | 送信内容 |
| :--- | :--- |
| チェックインの7日前 | スプレッドシート（またはVercel編集結果）に保存した「1週間前」の案内文 |
| チェックインの3日前 | チェックインガイドPDFのURL |
| チェックイン当日・12:00頃 | 玄関ドアの開け方（YouTube URL） |
| チェックイン当日・16:00頃 | 「チェックイン完了できましたか？」＋補足案内 |
| チェックアウト前日・12:00頃 | 明日チェックアウト・お忘れ物注意・清掃協力の案内 |

### 3.5 管理者画面（Vercel）でのテンプレート編集
- 「宿泊1週間前」に送るメッセージの文言を、Vercel上のフォームで編集できる。
- 保存時にGASのAPI（doPost）を呼び出し、受け取った文面をスプレッドシートの「通知テンプレート」用シート（または専用セル）に書き込む。
- 上記「チェックインの7日前」の自動送信時に、この保存済み文面を読み出してゲストに送信する。

## 4. データ構造仕様（Google Sheets）

### 4.1 想定シート構成（概要）
| シート名（例） | 用途 |
| :--- | :--- |
| **inquiries** | 問い合わせ履歴（LINE User ID, メッセージ, 受信日時） |
| **reservations** | 予約一覧（ゲストLINE User ID, チェックイン日, チェックアウト日, 金額, 確定日時, ステータス 等） |
| **guests** | ゲストマスタ（LINE User ID, 表示名, 初回登録日 等） |
| **templates** | 通知テンプレート（1週間前メッセージ本文 等） |
| **config** | 管理者LINE User ID, 清掃代・ペット時清掃代・1泊料金, Airbnb iCal URL, **月次PDF用 Drive フォルダID（driveFolderId）**, **清掃員LINEグループID（cleanerLineGroupId）**, チェックイン時刻, PDF/YouTube URL 等 |
| **photos** | 施設写真一覧（photoId, imageUrl, displayOrder, caption） |

※ 列定義の詳細は「データベース仕様書」に記載する。

### 4.2 その他ストレージ
- チェックインガイドPDF・YouTube URL は、スプレッドシートの `config` シートまたは `templates` シートにURL文字列で保持し、GASから参照する。

## 5. API仕様（GAS）

### 5.1 LINE Webhook（POST）
- **エンドポイント:** GASのWebアプリURL（Webhook用にデプロイしたもの）。
- **処理:** LINE Platform から送られるイベント（メッセージ等）を受信し、上記「問い合わせ受付」「管理者通知」を行う。応答は200 OKを返す。

### 5.2 管理者画面用API（Vercel → GAS）
- **メソッド:** POST
- **ボディ:** JSON。例: `{ "action": "saveTemplate", "data": { "key": "oneWeekBefore", "body": "編集したメッセージ本文" } }`
- **処理:** `action` に応じてスプレッドシートの該当シート／セルに保存する。CORS対応でJSONを返す。

### 5.3 共通レスポンス形式
- **成功:** `{ "status": "success", "result": { ... } }`
- **失敗:** `{ "status": "error", "message": "エラー内容" }`

## 6. 技術的制約・セキュリティ

### 6.1 通信
- **CORS:** VercelドメインからGASへのPOSTを許可する。GASは `ContentService` でMimeType.JSONを返す。
- **Webhook:** LINEの署名検証（Channel Secret を使用）を必ず実施する。

### 6.2 制限
- GASの実行時間制限（6分）、URL Fetch 呼び出し回数等は、LINE API の送信回数と合わせて設計する。
- 時間駆動トリガーは「1日1回」などでバッチ的に「今日送信すべき通知」を判定し、送信する方式を推奨する。

### 6.3 エラーハンドリング
- Webhook内で例外が発生した場合も、LINEに200を返し、エラー内容はログまたはスプレッドシートに記録する。
- 管理者画面からの保存APIでエラー時は、status: "error" と message を返し、画面側で表示する。

---

## 7. 運用・メンテナンス
- **GAS:** スクリプト修正後は「新しいデプロイ」でWebアプリおよびWebhook URLを更新する。
- **Vercel:** GitHubへのPushで自動ビルド・デプロイ。環境変数でGASのURL等を保持する。
- **管理者LINE User ID 変更:** スプレッドシートの `config` またはスクリプトプロパティを更新する。
