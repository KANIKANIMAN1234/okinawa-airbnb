## 1. テスト計画・概要
本テストの目的は、フロントエンド（Vercel）、バックエンド（GAS）、外部プラットフォーム（LINE/Google）の連携が、設計書通りの動作を維持しているかを検証することである。

### 1.1 テスト環境
* **ハードウェア:** iOS端末 (iPhone), Android端末
* **ソフトウェア:** LINEアプリ (最新版), Googleスプレッドシート
* **ネットワーク:** Wi-Fi環境, 4G/5G環境（モバイルデータ通信）

## 2. テストケース一覧

### 2.1 認証・初期登録機能 (Authentication)
| ID | テスト項目 | 操作手順 | 期待される結果 | 判定 |
| :--- | :--- | :--- | :--- | :--- |
| A-1 | 初回アクセス認証 | 初めてアプリを起動する。 | LINEの権限許可画面が表示され、許可後に「users」シートに新規行が追加されること。 | |

### 2.2 収支入力機能 (Input)
| ID | テスト項目 | 操作手順 | 期待される結果 | 判定 |
| :--- | :--- | :--- | :--- | :--- |
| B-1 | 収支タイプ切り替え | 「出金」⇔「入金」をタップ。 | ボタンの色が変わり、カテゴリの選択肢が即座に入れ替わること。 | |
| B-2 | 金額バリデーション | 金額を空欄にして「登録する」を押下。 | 「金額を入力してください」とアラートが出て、送信されないこと。 | |
| B-3 | 数値入力 | 金額に「1000」と入力して送信。 | スプレッドシートに数値の「1000」として正しく保存されること。 | |
| B-4 | 内容の自由記述 | 支払先に絵文字や特殊文字を入力。 | 文字化けせず、そのまま保存・表示されること。 | |
| B-5 | 二重送信防止 | 送信ボタンを連打する。 | 1回目の押下直後にボタンが非活性（グレーアウト）になり、複数登録されないこと。 | |

### 2.3 画像アップロード機能 (Image Upload)
| ID | テスト項目 | 操作手順 | 期待される結果 | 判定 |
| :--- | :--- | :--- | :--- | :--- |
| C-1 | カメラ起動テスト | 「カメラ起動」から写真を撮影。 | プレビューが表示され、正常に送信・保存されること。 | |
| C-2 | 複数枚登録 (上限) | 写真を3枚選択する。 | 3枚のプレビューが並び、追加ボタンがそれ以上反応しないこと。 | |
| C-3 | 写真の削除 | プレビュー右上の「×」を押す。 | 指定した画像のみが配列から消え、プレビューからも消えること。 | |
| C-4 | 大容量画像リサイズ | 高画質な（5MB以上）写真を選択。 | 送信前に自動圧縮され、GASのタイムアウトエラーにならずに保存されること。 | |



### 2.4 履歴閲覧・フィルタ機能 (History)
| ID | テスト項目 | 操作手順 | 期待される結果 | 判定 |

| D-2 | 人フィルタ | 家族Aの名前を選択。 | 家族Aが登録したデータのみが表示され、収支合計も再計算されること。 | |
| D-3 | 分類フィルタ | 「電気代」を選択。 | 電気代のレコードのみが抽出されること。 | |
