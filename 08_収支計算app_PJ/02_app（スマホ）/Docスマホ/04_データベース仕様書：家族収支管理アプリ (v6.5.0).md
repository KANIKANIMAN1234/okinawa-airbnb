## 1. データベース概要
本システムは、データベースとして「Googleスプレッドシート」を採用し、各シートを「テーブル（表）」として定義する。リレーショナルデータベースにおける「外部キー」の概念を LINE ID で実現し、データの関連付けを行う。



---

## 2. テーブル定義一覧
本システムは以下の2つの主要テーブルで構成される。

| 物理名 (シート名) | 論理名 | 用途 |
| :--- | :--- | :--- |
| **data** | 収支トランザクションテーブル | 全ての入金・出金記録と画像パスを保存する。 |
| **users** | ユーザーマスタテーブル | アプリを利用する家族メンバーの基本情報を管理する。 |

---

## 3. 各テーブル詳細設計

### 3.1 収支トランザクションテーブル (`data`)
本システムのメインデータ。日々の収支が1行1データ（1レコード）として蓄積される。

| 列 | 論理名 | 物理名 | データ型 | PK/FK | 必須 | 制約・備考 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| A | 登録日時 | timestamp | Date/Time | - | ◯ | GASにより `new Date()` で自動付与 |
| B | 利用日 | date | String (Date) | - | ◯ | YYYY-MM-DD 形式 |
| C | 利用者 | userName | String | - | ◯ | LINEプロファイル名 |
| D | 金額 | amount | Number | - | ◯ | 数値データ（カンマなしで保存） |
| E | 分類 | category | String | - | ◯ | プルダウン選択されたカテゴリ名 |
| F | 支払先 | shop | String | - | △ | 支払先または自由記述内容 |
| G | 画像URL | imageUrls | String (Text) | - | - | 最大3つのURLをカンマ(`,`)で結合 |
| H | LINE ID | lineId | String | FK | ◯ | ユーザーマスタ(users.lineId)と紐付け |
| I | 収支タイプ | type | String | - | ◯ | 「出金」または「入金」のいずれか |

### 3.2 ユーザーマスタテーブル (`users`)
システムの利用権限と、表示名の管理を行う。

| 列 | 論理名 | 物理名 | データ型 | PK/FK | 必須 | 制約・備考 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| A | LINE ID | lineId | String | PK | ◯ | LINE固有の `userId` (重複不可) |
| B | 名前 | name | String | - | ◯ | LINEの表示名 |
| C | 携帯番号 | tel | String | - | - | 予備項目。手動入力想定 |
| D | 登録日 | regDate | Date/Time | - | ◯ | 初回ログイン時の日時 |

## 4. 外部ストレージ連携 (Google Drive)
画像データはスプレッドシート内に直接保存せず、Googleドライブをバイナリストレージとして活用する。



* **保存先:** 指定された `FOLDER_ID` のGoogleドライブフォルダ。
* **ファイル命名規則:** `receipt_` + `タイムスタンプ` + `_連番` + `.jpg`
* **アクセス権:** 共有設定を「リンクを知っている全員（閲覧可）」に設定し、LIFFアプリ内からの参照を可能にする。

---

## 5. データ整合性・バリデーション規則
1.  **金額データ:** 入力時に文字列（全角や「円」など）が含まれている場合、フロントエンドの `Number()` 関数で数値変換し、純粋な数値として保存する。
2.  **カテゴリ名:** フロントエンドの `categories` 定数で定義された名称と完全に一致させる（フィルタリングの正確性を担保するため）。
3.  **日付形式:** 全ての集計ロジックが `YYYY-MM-DD` 形式を前提としているため、保存時に形式を統一する。
4.  **重複登録:** 履歴タブでのリロードや誤送信を防ぐため、送信後はフロントエンド側でボタンを非活性化する。

---

## 6. パフォーマンスとスケーリング
1.  **最大行数:** Googleスプレッドシートの制限（1,000万セル）に基づき、約50万行程度のデータを保持可能。
2.  **インデックス:** スプレッドシートにはDBのようなインデックス機能がないため、データ量が増大（数万行以上）した場合は、GAS側の `getList` 処理でフィルタリングを最適化するか、年ごとにシートを分割する「年次アーカイブ」を実施する。
