## 1. 概要
本資料は、「家族収支管理アプリ」のデータ保存先（スプレッドシート）、画像保存先（Googleドライブ）、およびAPIサーバー（GAS）を構築するための手順を定義するものである。

---

## 2. Googleスプレッドシートの作成と定義
システム上の「データベース」としての役割を果たすスプレッドシートを設定する。

1. **新規スプレッドシートの作成:** - [Googleスプレッドシート](https://sheets.new)にアクセスし、ファイル名を「家族収支管理DB」とする。
2. **シートの作成とヘッダー定義:** 以下の2つのシートを作成し、1行目に項目名を入力する。

### 2.1 dataシート (収支データ保存用)
| 列 | 項目名 (A1〜I1) | 備考 |
| :--- | :--- | :--- |
| A | 登録日時 | GASが自動付与 |
| B | 利用日 | アプリで選択した日付 |
| C | 利用者 | LINE名 |
| D | 金額 | 収支額 |
| E | 分類 | カテゴリ名 |
| F | 支払先 | 内容/店舗名 |
| G | 画像URL | ドライブURL (複数時はカンマ区切り) |
| H | LINE ID | ユーザー識別子 |
| I | 収支タイプ | 入金 or 出金 |



### 2.2 usersシート (ユーザーマスタ)
| 列 | 項目名 (A1〜D1) | 備考 |
| :--- | :--- | :--- |
| A | LINE ID | ユーザー識別子 (主キー) |
| B | 名前 | LINE名 |
| C | 携帯番号 | 予備項目 |
| D | 登録日 | 初回ログイン日時 |

3. **スプレッドシートIDの取得:** URLの `d/` と `/edit` の間にある長い英数字（ID）をコピーしてメモする。

---

## 3. Googleドライブ（画像保存先）の設定
アップロードされた領収書写真を保存するためのストレージを構成する。

1. **フォルダの作成:** Googleドライブ内に「家族収支管理_画像保存」フォルダを作成。
2. **フォルダIDの取得:** フォルダを開いた際のURL末尾の英数字（ID）をコピーしてメモする。
3. **共有設定:** - フォルダを右クリック ＞ [共有] ＞ [共有] を選択。
   - 一般的なアクセスを「**リンクを知っている全員**」に変更し、役割を「**閲覧者**」に設定する。
   - ※これにより、アプリ画面上で画像を表示可能にする。

---

## 4. Google Apps Script (GAS) の構築・デプロイ
フロントエンドからのリクエストを処理するAPIサーバーを構築する。

### 4.1 スクリプトの作成
1. スプレッドシートのメニューから [拡張機能] ＞ [**Apps Script**] を開く。
2. デフォルトの `myFunction` を全て削除し、提供された `Code.gs` を貼り付ける。

### 4.2 設定値の書き換え
コード内の冒頭にある2つの定数を、先ほどメモしたIDに書き換える。
```javascript
const SPREADSHEET_ID = "ここにスプレッドシートIDを貼り付け";
const FOLDER_ID = "ここにフォルダIDを貼り付け";
```

### 4.3 ウェブアプリとしてのデプロイ (最重要設定)
プログラムを外部（VercelやLINE）から呼び出し可能な「APIサーバー」として公開する手順です。

1. **デプロイメニューの起動:** GASエディタ画面右上の [**デプロイ**] ボタンをクリックし、[**新しいデプロイ**] を選択。
2. **種類の選択:** 左側の「種類の選択（歯車アイコン）」をクリックし、「**ウェブアプリ**」を選択。
3. **詳細設定の入力:**
   - **説明:** `家族収支管理API v6.5.0`（バージョン管理しやすくするため明記推奨）
   - **次のユーザーとして実行:** 「**自分**」を選択。
   - **アクセスできるユーザー:** 「**全員**（Anyone）」を必ず選択。
     - *注意: 「自分のみ」や「Googleアカウントを持つ全員」にすると、LINEからのアクセスが拒否されます。*
4. **デプロイの実行:** [デプロイ] ボタンをクリック。



5. **アクセスの承認:** - 「アクセスを承認」というポップアップが出たら、自分のGoogleアカウントを選択。
   - 「このアプリは Google で確認されていません」と警告が出た場合は、左下の [**詳細**] ＞ [**家族収支管理（安全ではないページ）に移動**] をクリック。
   - 最後に [**許可**] をクリック。
6. **URLの取得:** 発行された「**ウェブアプリ URL**」（末尾が `/exec` のもの）をコピーし、メモ帳等に保存。
   - *注意: このURLは Vercel の `index.html` 内にある `GAS_URL` 変数に貼り付ける必要があります。*

## 5. 運用上の注意点とメンテナンス

### 5.1 コード修正後の「再デプロイ」ルール (最頻出トラブル)
GASのコードを書き換えて保存（Ctrl+S）しただけでは、ウェブアプリには反映されません。
- 修正を反映させるには、[デプロイ] ＞ [デプロイを管理] ＞ [編集（鉛筆アイコン）] ＞ [バージョン：**新バージョン**] を選択して、再度「デプロイ」を押し直す必要があります。

### 5.2 Googleドライブの容量管理
- **ストレージ監視:** 保存された写真はGoogleアカウントの全体容量（15GB）を消費します。
- **定期整理:** 不要になった古い写真をドライブのフォルダから削除しても、スプレッドシートのURLリンクは自動では消えません。不整合を防ぐため、スプレッドシートの行ごと削除する運用を推奨します。



### 5.3 スプレッドシートの列の型設定
- **金額列（D列）:** スプレッドシート側で「書式」＞「数字」＞「通貨（または数値）」に設定してください。
- **日付列（B列）:** 同様に「書式」＞「数字」＞「日付」に設定することで、スプレッドシート上での並べ替えやグラフ化が正確になります。

## 6. 接続テストと疎通確認

構築が完了したら、以下の手順で正しく動作しているか確認してください。

1. **API単体テスト:** - メモした「ウェブアプリ URL」をブラウザのアドレスバーに貼り付け、Enterキーを押す。
   - 画面に `"GAS API v6.1.0 active."`（またはバージョン名）と表示されれば、APIサーバーとして正常に待機しています。
2. **スプレッドシート連携テスト:**
   - アプリ（LINE/Vercel）からダミーデータを送信。
   - スプレッドシートの `data` シートに、即座に新しい行が追加されるか確認。
3. **画像保存テスト:**
   - アプリから写真を撮影して送信。
   - Googleドライブの指定フォルダに画像ファイルが生成され、かつスプレッドシートのG列に正しいURLが書き込まれているか確認。
