## 4. インターフェース詳細 (Vercel ⇔ GAS)

### 4.1 通信プロトコル
* **メソッド:** `POST`
* **データ形式:** `application/json`
* **文字コード:** `UTF-8`

### 4.2 リクエストパラメータ
全リクエストは以下の共通構造を持つ。

```json
{
  "action": "アクション名",
  "data": { ...各機能固有のペイロード... }
}```

### 4.3 レスポンスパラメータ (Response Format)
全アクション共通で、GASバックエンドから以下のJSON構造で返却される。

| フィールド名 | データ型 | 内容 |
| :--- | :--- | :--- |
| **status** | String | 処理結果ステータス（`"success"` または `"error"`） |
| **result** | Object | 成功時の返却データ（リストデータ、メッセージ、ユーザー名配列など） |
| **message** | String | エラー発生時の詳細なエラーメッセージ（成功時は空文字） |

## 6. セキュリティ・制約事項

### 6.1 CORS (Cross-Origin Resource Sharing) 対策
Vercelドメイン（フロントエンド）からGoogleドメイン（GAS）へのクロスドメイン通信を許可するため、GAS側の `doPost(e)` 内で以下の処理を徹底する。

* **ContentServiceの利用:** `ContentService.createTextOutput` を用いて、MimeTypeを `JSON` に指定してレスポンスを返す。これにより、ブラウザの同一生成元ポリシー（Same-Origin Policy）を回避し、安全なデータ交換を行う。

### 6.2 画像データのエンコーディング
画像ファイルはバイナリ形式のまま送信せず、以下の手順で連携する。

1.  **エンコード:** フロントエンドで `Canvas API` を用いて画像をJPEG圧縮し、`Base64 (DataURL)` 文字列に変換。
2.  **ペイロードへの格納:** JSONオブジェクトの1つのプロパティ（`imageBlobs` 配列）として送信。
3.  **デコード:** GAS側で `Utilities.base64Decode` を実行し、バイナリ（Blob）に戻してGoogleドライブへ保存。

### 6.3 流量制限 (Quotas) とキャパシティ
Google Workspace および LINE LIFF の公式制限に基づき、以下の制約内で運用する。

* **GAS URL Fetch:** 20,000回 / 日（Googleアカウント無料枠）。
* **Google Drive ストレージ:** 15GB（標準アカウント全体容量）。
* **画像リサイズ制約:** 通信エラー防止のため、1画像あたり最大横幅 1024px、画質 0.7（70%）の制限をフロントエンド側で強制する。
